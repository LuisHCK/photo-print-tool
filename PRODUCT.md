# Product

## Register

product

## Platform

web

## Users

Primary — home users and hobbyist photographers printing personal photos (family, events, travel, keepsakes). They want precise results without learning design software or doing manual math.

Secondary — print-shop operators and serious photographers preparing batch print jobs. They need exact, repeatable layouts and unit-accurate dimensions in a predictable tool.

## Product Purpose

Exact, repeatable photo print layouts without complex design software. Everything runs client-side in the browser — no uploads, no accounts, no data leaves the machine.

## Positioning

The familiar Windows image print wizard, rebuilt for modern browsers with serious print control. Same quick-print flow, zero learning curve — but with real unit precision, custom layouts, and the depth power users need.

## Brand Personality

Precise, warm, approachable. The tool feels like a well-made craft tool — accurate enough for a professional, friendly enough for anyone to pick up.

## Anti-references

No generic SaaS look. Avoid the striped-header, sidebar-heavy dashboard aesthetic common to admin tools. This is a focused utility, not a data dashboard. Also: not the Windows XP/7 aesthetic — modern, clean, warm. Not Canva/Photoshop — no creative-suite complexity for a print utility.

## Design Principles

- Trust the defaults. Smart presets and sensible defaults so most users never need to tweak; advanced controls available but never in the way.
- Save paper first. The default layout minimizes pages by computing the tightest margins and optimal grid. Room to breathe is optional; efficiency is the default.
- Results over chrome. Every pixel serves the print result. Decorative UI that doesn't help the user lay out photos is noise.
- Practice what you preach. A layout tool demonstrates good layout itself — spacing, alignment, clarity.
- One job, done well. Print layout and nothing else. No feature creep into photo editing, sharing, or storage.

## Accessibility & Inclusion

Standard WCAG 2.1 AA good practices: contrast, keyboard navigation, screen reader support for controls. Reduced motion support via `prefers-reduced-motion`.
